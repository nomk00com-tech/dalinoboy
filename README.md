# PUESC/SENT GPS Monitor Bot

Telegram-бот для мониторинга GPS-трекеров через польский портал PUESC/SENT (форма RMPD-406).

## Структура проекта

```
.
├── checker.py       # Фаза 1 — автономный чекер (запускать первым)
├── main.py          # Точка входа бота
├── handlers.py      # Telegram-обработчики
├── puesc.py         # Playwright-автоматизация PUESC
├── scheduler.py     # Периодические проверки (APScheduler)
├── db.py            # SQLite операции
├── requirements.txt
├── .env.example     # Шаблон переменных окружения
└── bot.log          # Лог (создаётся автоматически)
```

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
playwright install chromium
```

### 2. Создать .env

```bash
cp .env.example .env
# Отредактируйте .env — заполните все переменные
```

### 3. Фаза 1 — протестировать чекер

Перед запуском бота убедитесь, что чекер работает:

```bash
python checker.py --rmpd RMPD123456 --plate ABC1234 --tracker LOC-987654
```

Или используйте тестовые данные из `.env` (TEST_RMPD / TEST_PLATE / TEST_TRACKER):

```bash
python checker.py
```

Ожидаемый вывод:
```
STATUS  : signal_ok
LAST POS: 2024-03-15 14:22:05
MESSAGE : GPS OK. Last position: 2024-03-15 14:22:05.
```

### 4. Запустить бот

```bash
python main.py
```

## Логика уведомлений (с учётом движения)

Главное требование заказчика: **GPS обязателен только во время движения**. На стоянке
локализатор может законно молчать (водитель вправе выключить GPS) — за это не штрафуют,
и бот **не должен слать тревоги во время парковки**. Штраф начинается после ~60 минут
без сигнала **в движении**.

Поэтому бот **различает движение и стоянку** по изменению GPS-координат между проверками:

| Ситуация | Тревога? |
|----------|----------|
| Сигнал есть, координаты обновляются | ✅ всё ок |
| Сигнал пропал, но авто **стояло** (координаты не менялись) → стоянка | 🅿️ нет тревоги |
| Сигнал пропал, авто **двигалось**, прошло < `ALERT_MINUTES` | ⏳ ждём (пока без тревоги) |
| Сигнал пропал в движении ≥ `ALERT_MINUTES` (по умолч. 50 мин) | ⚠️ **тревога** всем |
| Оба трекера авто в тревоге одновременно | 🚨 **критично** всем |
| Сигнал восстановился (после тревоги) | ✅ уведомление всем |
| `błąd 200` (PUESC: локализатор не передаёт) | трактуется как «нет сигнала» |
| Неверные данные RMPD/номер/трекер | ❓ только админам, один раз |
| PUESC недоступен | 🔴 только админам, один раз за сбой |

**Как определяется движение:** бот хранит координаты каждой проверки. Если за последние
несколько проверок (пока сигнал был) позиция сместилась более чем на `MOVE_THRESHOLD_M`
метров (по умолч. 500 м) — авто ехало. Если координаты «застыли» в одной точке — стоянка.

**Анти-спам:** тревога/восстановление шлются только в момент СМЕНЫ состояния
(нет-тревоги → тревога и обратно), без повторов между проверками.

### Параметры в .env

| Переменная | Смысл | По умолч. |
|------------|-------|-----------|
| `CHECK_INTERVAL_MINUTES` | как часто проверять GPS | 5 |
| `SIGNAL_FRESH_MINUTES` | старше скольких минут позиция = «не передаёт сейчас» | 15 |
| `ALERT_MINUTES` | сколько минут потери сигнала **в движении** до тревоги | 50 |
| `MOVE_THRESHOLD_M` | сдвиг (м) между проверками, считающийся движением | 500 |
| `NEWS_INTERVAL_MINUTES` | как часто проверять новости PUESC | 30 |
| `NEWS_CATEGORY_URL` | URL категории новостей SENT (менять не нужно) | категория 603451585 |

## Новости PUESC (аварии SENT-GEO)

PUESC иногда публикует в разделе [Aktualności](https://puesc.gov.pl/aktualnosci)
объявления об **авариях системы SENT-GEO**: когда сервер не работает, движение
без отслеживания может быть временно разрешено; когда аварию устранили —
отслеживание снова обязательно.

Бот **автоматически следит** за категорией новостей «Przewóz towarów objęty
monitorowaniem» (только релевантные SENT/мониторинг новости) каждые
`NEWS_INTERVAL_MINUTES` минут:

- Новая статья → уведомление всем получателям с заголовком, датой и ссылкой.
- Объявления об **аварии / недоступности** (`awaria`, `niedostępność`,
  `wstrzymanie`, `przywrócenie`) помечаются 🔴 как важные.
- При первом запуске бот запоминает текущие статьи без рассылки (без спама старым).
- Кнопка **📰 Новини PUESC** в боте показывает последние новости по запросу.

Дедупликация — по `articleId` (таблица `news_seen`), повторов нет.

## Важная оговорка об автоматизации PUESC

**Регистрация/логин НЕ нужны.** Форма RMPD-406 / SENT-406 — публичная и бесплатная:
официально *«Usługa jest dostępna bez logowania»*. Бот заполняет 3 поля и читает ответ.

Форма всё равно **динамическая** (Liferay рендерит её через JavaScript), поэтому
используется браузер (Playwright), а не обычный HTTP-запрос.

Данные о позиции появятся, **только если**:
- зголошение SENT активно (не закрыто/не истекло);
- зголошение полное (содержит данные перевозчика);
- локализатор реально передал геоданные в SENT-GEO.

### Селекторы и парсинг — проверены на живой форме (2026-06)

Селекторы формы стабильны и подтверждены реальным запросом:

| Поле | Селектор |
|------|----------|
| Номер RMPD | `#_Sent_Rmpd406Portlet_rmpdNumber` |
| Номер авто | `#_Sent_Rmpd406Portlet_truckNumber` |
| Локализатор | `#_Sent_Rmpd406Portlet_geoLocatorNumber` |
| Кнопка «Zatwierdź» | `[data-sent-role='command-submit']` (id меняется каждую сессию — берём по data-атрибуту) |

Как PUESC отвечает (проверено) и как мы это трактуем:

| Ответ PUESC | Статус бота |
|-------------|-------------|
| `RMPD416 …`, `Status zgłoszenia: Kompletne`, координаты + `Czas: …` свежий | `signal_ok` |
| то же, но `Czas` старше `MAX_SIGNAL_AGE_HOURS` | `signal_missing` |
| `RMPD416 …`, но нет координат (локализатор не шлёт в GEO) | `signal_missing` |
| `błąd 200` — PUESC: локализатор не передаёт сигнал | `signal_missing` |
| alert `Błąd: … not found` | `invalid_data` |
| alert `Błąd: … niezgodny ze schematem XSD` | `invalid_data` (неверный формат) |
| таймаут/сеть | `site_error` |

**Возраст позиции** считается относительно времени, которое сама страница печатает
(`Aktualna data i czas (strefa czasowa Warszawa)`) — это убирает проблемы часовых поясов.

### Формат номера локализатора

PUESC проверяет номер по XSD-маске:
```
[Z|U|M] + 2 цифры − 2 буквы 2 цифры 2 буквы − 1 цифра      например: Z21-AF67XZ-8
```
Бот сам отбраковывает номера неверного формата ещё до запроса к PUESC (см. `LOCATOR_RE` в `puesc.py`).
У одного RMPD может быть **основной и запасной** локализатор — каждый проверяется отдельным запросом.

### Отладка (видимый браузер)

```bash
python checker.py --headed --rmpd RMPD20260607000433 --plate BC8849PO --tracker Z21-AF67XZ-8
```
Флаг `--headed` (или `PUESC_HEADLESS=0` в `.env`) откроет окно браузера.

## Модель данных

```sql
users       — telegram_id, name, is_admin, notifications_enabled
vehicles    — id, name, plate_number
trackers    — id, vehicle_id, provider, tracker_number
trips       — id, vehicle_id, rmpd_number, status, created_at, finished_at
trip_checks — id, trip_id, tracker_id, status, last_position_time,
              latitude, longitude, alarm, raw_message, created_at
```
`latitude`/`longitude` нужны для определения движения, `alarm` — флаг активной
тревоги (для анти-спама: уведомление шлётся только при смене флага).
